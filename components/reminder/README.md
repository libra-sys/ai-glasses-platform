# 定时提醒组件

## 功能描述

定时提醒组件通过组件化开关机制实现自动提醒功能。安装组件后，服务器会根据配置的时间自动播报提醒内容，无需额外界面设置。

## 核心特性

- **组件驱动**: 通过安装/卸载组件控制提醒功能开关
- **服务器控制**: 提醒任务由服务器后台定时执行
- **自动播报**: 到点自动通过语音播报提醒内容
- **低功耗**: 眼镜端仅接收提醒通知，无需本地定时器

## 配置说明

### 组件配置格式

```json
{
  "featureType": "reminder",
  "reminders": [
    {
      "time": "08:30",
      "text": "早上吃药"
    },
    {
      "time": "12:00",
      "text": "午餐时间"
    },
    {
      "time": "22:00",
      "text": "准备休息"
    }
  ]
}
```

### 字段说明

- `featureType`: 固定为 `"reminder"`，标识为定时提醒组件
- `reminders`: 提醒任务列表
  - `time`: 提醒时间，格式为 `HH:MM`（24小时制）
  - `text`: 提醒内容文本

## 使用方式

1. 在网站发布组件时，配置 `config.featureType = "reminder"`
2. 在 `config.reminders` 中添加提醒任务
3. 小程序同步并安装组件到服务器
4. 服务器自动启动定时提醒引擎
5. 到点自动播报提醒内容

## 卸载组件

在小程序中卸载组件后，服务器会自动：
- 关闭定时提醒功能
- 清空提醒任务列表
- 停止后台定时检查

## 硬件要求

- ESP32 开发板
- TFT 显示屏（可选）
- WiFi 连接

## 技术实现

组件通过以下机制实现：

1. 服务器端：
   - `reminder_enabled`: 全局开关
   - `reminder_jobs`: 提醒任务列表
   - `reminder_loop()`: 后台定时检查任务
   - `apply_component_effects()`: 安装时启用
   - `remove_component_effects()`: 卸载时关闭

2. 眼镜端：
   - 周期性轮询服务器状态
   - 接收并显示提醒内容
   - 可选语音播报

## 注意事项

- 提醒时间采用24小时制
- 每个提醒每天只触发一次
- 服务器需保持运行状态
- WiFi 连接需稳定
